Uji Coba 2



Jika semua radio button memiliki atribut "name" yang sama, hanya satu pilihan yang bisa dipilih.
Jika setiap radio button memiliki atribut "name" yang berbeda, maka bisa dipilih secara bersamaan.

Uji Coba 3




Checkbox memungkinkan pemilihan lebih dari satu opsi, berbeda dengan radio button yang hanya memungkinkan satu pilihan.
Jika atribut name dibuat sama, ini berpengaruh saat form dikirim, karena semua pilihan dikumpulkan dalam satu array data.
Jika name berbeda, setiap checkbox dikirim sebagai field terpisah dalam form.

Uji Coba 4




Kesimpulan:
1. Input type="color" memungkinkan untuk memilih warna dari palet warna.
2. Input type="date" menyediakan untuk memilih tanggal yang bervariasi tergantung pada browser. Pengguna bisa memilih tanggal dengan mudah.
3. Input type="button" digunakan untuk membuat tombol yang bisa mengeksekusi fungsi JavaScript. Dalam contoh ini, tombol menampilkan alert "Hello World!".

Uji coba 5




Kesimpulan:
1. Input type datetime-local memungkinkan untuk memilih tanggal dan waktu secara bersamaan.
2. Input type month memungkinkan untuk memilih bulan dan tahun tanpa memilih tanggal spesifik. Tidak semua browser mendukung tampilan yang sama.
3. Input type email digunakan untuk memasukkan alamat email yang valid. Browser modern memvalidasi format email sebelum formulir dikirim.

Uji coba 6




Kesimpulan:
1. Input type image berfungsi sebagai tombol submit berbentuk gambar. Saat diklik, form akan dikirim, mirip dengan tombol submit biasa.
2. Input type file memungkinkan untuk memilih file dari perangkat mereka untuk diunggah. Browser akan menampilkan tombol "Choose File".
3. Input type hidden digunakan untuk menyimpan nilai tersembunyi yang tidak terlihat oleh pengguna. Nilai ini akan dikirim ke server saat formulir dikirim.

Uji coba 7



Kesimpulan:
1. Input type= number memungkinkan untuk memasukkan angka dalam rentang tertentu menggunakan tombol panah atas/bawah atau mengetik secara manual.
Jika angka yang dimasukkan melebihi batas (min=1, max=10), browser bisa menampilkan peringatan saat form dikirim.
2. Input type= range menampilkan slider untuk memilih nilai dalam rentang tertentu.
3. Jika min=-50 pada input number maka Pengguna bisa memasukkan angka mulai dari -50 hingga 10, yang berarti batas bawah berubah sesuai nilai min.
4. Jika min=-50 pada input range maka Slider akan bisa bergerak dari -50 hingga 50, mengubah batas bawah yang bisa dipilih.

Uji coba 8




Kesimpulan:
1. Input type=url Hanya untuk menerima format URL yang valid.
2. Input type=search Sama seperti input teks, tetapi dengan fitur tambahan seperti tombol "X" untuk menghapus teks di beberapa browser.
3. Input type=tel digunakan untuk memasukkan nomor telepon. Pattern [0-9]{3}-[0-9]{2}-[0-9]{3} berarti pengguna harus memasukkan format 123-45-678.

Uji coba 9



Kesimpulan:
1. Input type=time memungkinkan pengguna untuk memilih waktu (jam dan menit) tanpa tanggal.
2. Input type=week memungkinkan pengguna untuk memilih minggu dalam satu tahun.
